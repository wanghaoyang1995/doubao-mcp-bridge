
### 背景
豆包桌面版系统集成很强: 全局AI划词翻译/解释、OCR文字提取、全局实时字幕、实时语音通话... -- 但未开放MCP能力, 这是个巨大的遗憾
Cursor/Trae: 自由度高, 但系统集成弱, 定位是AI编程开发工具, 并非日常工作助手

需求：我需要一个有MCP能力的桌面工作助手

### 虚拟MCP调用方案
```
┌───────────────────────────────────────────────────────┐
│               豆包桌面版(目前MCP未开放)                  │
│  1.接收提示词 → 知道可用工具 & 调用格式                   │
│  2.输出 MCP_CALL_JSON: {"server":"xxx","tool":"xxx"}  │
│  3.接收 MCP_RESULT_JSON 并整理回答                      │
└───────────────▲───────────────┬──────────────────────┘
                │               │
                │ 剪贴板中转      │
                ▼               ▼
┌─────────────────────────────────────────────────┐
│              AutoHotkey (AHK 快捷键层)            │
│ F9: 调用 bridge.py prompt → 生成工具提示词粘贴      │
│ F10: 调用 bridge.py call → 读取剪贴板指令执行       │
└───────────────▲───────────────┬─────────────────┘
                │               │
                │               │
                ▼               ▼
┌─────────────────────────────────────────────────┐
│            Python bridge.py 桥接脚本              │
│ ├─ prompt: 读取servers.json → 获取所有MCP工具      │
│ └─ call: 解析MCP_CALL_JSON → 调用MCP → 返回结果    │
└───────────────▲───────────────┬─────────────────┘
                │               │
                ▼               ▼
┌─────────────────────────────────────────────────┐
│               MCP Server(示例: anki-mcp)         │
│ 对接本地软件: Anki / 文件系统 / 浏览器等             │
└─────────────────────────────────────────────────┘
```

### 使用

1. 安装：`pip install mcp pyperclip`，安装 Node.js、AutoHotkey

2. 配置MCP服务：`servers.json`，例如：

```json
{
  "mcpServers": {
    "anki-mcp": {
      "command": "cmd",
      "args": ["/c", "npx", "-y", "@ankimcp/anki-mcp-server", "--stdio"],
      "env": {
        "ANKI_CONNECT_URL": "http://localhost:8765"
      }
    }
  }
}
```

3. 启动 AHK 脚本（双击hotkey_mcp.ahk）

4. 首次初始化 Prompt, 在豆包输入框中按 F9

5. 触发 MCP 调用

当调用MCP时，豆包会输出 MCP_CALL_JSON 调用标记，复制该内容（手动）并将光标定位到豆包聊天输入框:

按 F10，bridge.py 会自动: 读取剪贴板、调用 MCP、执行 tool、将结果写回剪贴板并发送到豆包会话，豆包继续执行后续流程


### 总结
1. 自动化环节可进一步优化，如自动查找豆包软件界面最后一条回复控件、文字输入控件，实现全自动调用
2. 豆包的虚拟MCP能力是足够的, prompt设计很重要
3. 目前一次只调用一个MCP工具, 理论上该方案多阶段任务编排、多任务并行调用也是可行的, 工程实现复杂一些
4. 希望未来豆包开放MCP, 就不必折腾了
